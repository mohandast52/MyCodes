        • only perfect squares has odd number of divisors! BUT WHY & HOW?
          - Well, divisors of any number exist in pairs. (eg) if we're looking for divisors of 12, then the pairs would be (1,12) ,(2,6) ,(3,4) ie, Even.
          However if the number is a perfect square say 16 then the divisors would be (1,16) , (2,8) , (4,4) .
          We see there's a divisor ( sqrt(16) ) which appears twice. Hence we'd subtract 1 of its occurrence which makes the total count odd.

Optimization problem are those which requires minimum or maximum result! principle of optimality! in greedy, we take decision once and follow that, but in dynamic programming we take decision on every stage
          Top down : memoization uses top down, which is recursive calls!
          Bottom up : value filled from bottom to top i.e from 0 to n.
          example : fibonacci in recursive, fib(n) => fib(n-1) & fib(n-2); ie. from top its calling subproblems which is going downwards. Hence top-down approach! in DP, fib[i] = fib[[i-2] + fib[n-1]; here we are filling table from 0 to n and using that stored values (tabulation) i.e bottom to top;
